D0 & D1 — Serial pins (UART) · सिरियल पिन

डिजिटल ० र १ = कम्प्युटरसँग कुराकानी

Pins D0 and D1 are reserved for hardware Serial communication (UART). They let the Arduino talk to your computer—or to another device such as a Bluetooth or GPS module—one bit at a time.

PinNameDirectionWhat it doesWhy it matters
D0 RX (Receive) Input to Arduino Listens for data coming into the board Connected inside the Uno to the USB chip so your PC can send sketches and Serial Monitor commands
D1 TX (Transmit) Output from Arduino Sends data out from the board Used when you Serial.print() — text goes to the computer’s Serial Monitor
Think of it like a phone call · फोन कल जस्तै TX (D1) = you speaking (Arduino sends). RX (D0) = you listening (Arduino receives). For two Arduinos to talk, wire TX → RX and RX → TX crosswise, and share GND.

When you upload code or use Serial Monitor

While the USB cable is connected, the computer uses D0/D1 to upload your program and for debugging with Serial.begin(9600) and Serial.println(). If you also connect sensors, motors, or LEDs to D0 or D1, those wires can fight the USB chip—uploads may fail or Serial output may look like garbage.

Rules for this robotics course · नियमहरू

  • Do not wire components from this site to D0 or D1 unless you are doing a dedicated Serial project and know how to disconnect USB conflicts.
  • Use D2–D13 for LEDs, sensors, servos, relays, and motor drivers instead.
  • After upload, you can still use Serial.print() in code—it uses D1 automatically; you do not need extra wires to the USB port.
// Example: print sensor value to computer (uses D1 TX internally)
void setup() {
  Serial.begin(9600);  // 9600 bits per second
}

void loop() {
  Serial.println("Hello from Arduino!");
  delay(1000);
}

Safety · सुरक्षा Do not run motors or relays directly from Arduino pins — use a driver (L298N) or relay module. Share one GND (ग्राउन्ड) between Uno and external power supplies. Uno 5V pin can supply only ~400 mA total — not enough for motors.
